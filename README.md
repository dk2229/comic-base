# Comic Base (漫画管理Webアプリケーション)

## 概要
職業訓練校での開発演習で作成した、漫画情報を管理できるWebアプリケーションです。
好きな漫画作品の情報、キャラクター、レビューを管理し、特定作品（HUNTER×HUNTER）のクイズ機能も楽しめます。

Spring Bootなどのフレームワークを使用せず、Javaの基礎技術（Servlet/JSP）のみで構築することで、Webアプリケーションの裏側の仕組み（HTTPリクエストやMVCの流れ）を深く理解することに重点を置きました。

## 主な機能
* **ユーザー管理**: 会員登録・ログイン・編集・削除
* **漫画管理**: 漫画情報の登録・編集・削除・一覧表示（CRUD機能）
* **レビュー機能**: 5段階評価付き感想投稿
* **掲示板機能**: 匿名での自由な書き込み・アンカー機能
* **キャラクター図鑑**: キャラクター情報の管理
* **マイページ**: 個人の投稿履歴管理
* **クイズ機能**: 漫画の読み方クイズ（全10問ランダム出題）

## 💡 開発方針（AI活用と技術選定について）
本アプリケーションは、開発効率化のために生成AI（Claude）を補助として活用しましたが、自身の技術力向上のため、以下の厳格なルールを設けて開発しました。

1.  **ブラックボックス化の排除**
    * 学習していない言語（JavaScriptなど）や、説明できないライブラリの使用を禁止しました。
    * AIが生成したコードは必ず一行ずつ読み解き、ロジックを完全に理解してから実装しました。
2.  **設計パターンの統一**
    * 既存の機能（会員登録機能など）のコードを「ひな形」としてAIに提示し、「これと同じMVCパターンで作って」と指示することで、全ての機能で統一された構造を維持しました。
    * これにより、バグ発生時も原因特定が容易になり、高い保守性を実現しました。

## 技術的なこだわり
1.  **統一されたMVCパターンの実装**
    * 全機能で `Controller` → `Service` → `DAO` の一貫した処理フローを徹底しました。
2.  **DAOパターンの活用**
    * データベース操作をDAOクラスに集約し、SQL処理とビジネスロジックを明確に分離しました。
3.  **複雑な日本語文字正規化処理**
    * クイズ機能において、ユーザーの回答（ひらがな/カタカナ）の表記ゆれを吸収する処理を実装しました。
4.  **セキュリティを考慮したセッション管理**
    * ログイン状態の保持や、権限のないアクセスの制御を行っています。

## 開発で学んだこと
* **MVCアーキテクチャの実践的理解**: 役割分担を意識したコーディングの重要性を学びました。
* **データベース設計とCRUD操作**: テーブルのリレーション設計からSQLによるデータ操作まで一通り実装できるようになりました。
* **エラーハンドリング**: 404/500エラーの原因特定と修正（デバッグ）のプロセスを経験しました。
* **Git/GitHub**: チーム開発を見据え、ソースコード管理の基礎を独学で習得しました。
